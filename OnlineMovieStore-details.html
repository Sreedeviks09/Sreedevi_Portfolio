Online Movie Store

Online Movie Store with Django The Online Movie Store is a web application designed to provide users with a seamless experience of browsing, searching, and purchasing movies. This platform is built using the Django framework, combined with responsive front-end technologies like HTML, CSS, and Bootstrap for a user-friendly interface. The database management system ensures secure storage and retrieval of movie and user data.

Technologies Used: Backend: Python, Django Framework Frontend: HTML, CSS, Bootstrap Database: Admin(Super User) Version Control: Git/GitHub (for version control and project collaboration)

Creation: Users can add new tasks with details like task name, priority, and due date. A form is provided for users to input task details, which are then stored in a database. Listing: All the tasks are displayed on the homepage. Each task displays its name, priority, and due date. Update: Users can update existing tasks by modifying the task name, priority, or due date. A user-friendly form is provided for editing tasks, with pre-filled values that can be updated. Deletion: Users can delete tasks they no longer need. A confirmation page is shown before deletion to prevent accidental removal of tasks. Management: CRUD operations are implemented (Create, Read, Update, Delete) allowing the user to manage tasks effectively. The app ensures that the user’s task data is safely saved in the database and can be retrieved whenever required.

Project Structure: views.py: Contains the logic for handling requests and rendering the appropriate HTML templates. Views for adding a new movie (Movieadd), deleting a movie (delete), and updating a movie (update). urls.py: URL routing defines how requests are mapped to views (URLs for home, movie deletion, movie update). models.py: Defines the moviestore model with fields like name, priority, and date that represent a task's core attributes. forms.py: A Django ModelForm is used to handle movie creation and updates via a form interface. templates: home.html: Displays a list of movie and a form to add new movie. delete.html: Displays a confirmation form before deleting a movie. edit.html: Provides a form to update an existing movie. base.html: Base template containing common HTML structure like navigation, header, and footer. Static files: CSS (style.css): Custom styles to make the application visually appealing.

Key Learning Outcomes: Django Fundamentals: Learned how to build a web application with Django’s MVT (Model-View-Template) architecture. Mastered Django models, forms, views, and templates for creating dynamic web applications. Database Handling: Used Django’s ORM (Object-Relational Mapping) for database operations, including adding, updating, and deleting records. Demonstrated the ability to integrate the app with a relational database (SQLite). Frontend Development: Created interactive and responsive forms using HTML and Bootstrap for the user interface. Integrated custom styles in CSS for better UI/UX. Version Control: Used Git for version control and GitHub for managing and sharing the code.

Features to Improve or Add (Future Work): User Authentication: Add user authentication so that users can register and log in to manage their movie. Each user will only have access to their own movie. Movie Categories and Filters: Add categories for movies (Work, Personal, etc.) and allow users to filter title by category, priority, or due date. Movie Notifications: Add notifications or email alerts to remind users about tasks that are due soon. Movie Status: Implement task completion status (e.g., Pending, Completed) and allow users to mark tasks as complete.

This project demonstrates expertise in full-stack web development with Django, focusing on e-commerce functionality and responsive design. It showcases skills in back-end development, front-end design, and database management.
